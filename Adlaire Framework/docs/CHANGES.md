## Ver.1.13-15 — 静的ファイル配信廃止

**日付**: 2026-04-11
**種別**: 廃止

- `middleware.ts` の静的ファイル配信（`serveStatic()`）を廃止

## Ver.1.12-14 — バグ修正（56件精査・10件修正）

**日付**: 2026-04-10
**種別**: バグ修正

全56項目精査（重大3件・中程度4件・軽微3件）

| # | 深刻度 | 対象ファイル | 概要 |
|:-:|:------:|------------|------|
| 1 | 重大 | `middleware.ts` | CORS credentials + wildcard origin RFC 違反を修正 |
| 2 | 重大 | `middleware.ts` | rateLimit エントリの定期クリーンアップ未実装によるメモリリーク修正 |
| 3 | 重大 | `server.ts` | SSE data マルチライン未対応（RFC 8895 違反）を修正 |
| 4 | 中程度 | `middleware.ts` | Cookie name の不正な encodeURIComponent を修正 |
| 5 | 中程度 | `server.ts` | ファイル配信レスポンスの Content-Disposition filename 未エスケープを修正 |
| 6 | 中程度 | `server.ts` | サーバー停止処理の二重呼び出しを修正 |
| 7 | 中程度 | `middleware.ts` | jwtAuth の algorithms オプションが実際の検証に使用されていない問題を修正 |
| 8 | 軽微 | `server.ts` | 環境変数スキーマで空文字列が number 0 に暗黙変換される問題を修正 |
| 9 | 軽微 | `server.ts` | フォームデータの同名フィールドが上書きされる問題を修正（配列として返す） |
| 10 | 軽微 | `middleware.ts` | 圧縮済みレスポンスに二重圧縮が適用される問題を修正 |
