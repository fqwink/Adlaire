| バージョン | 日付 | 変更内容 |
|-----------|------|---------|
| Ver.2.5 | 2026-04-11 | **型安全をアーキテクチャによる構造的保証に再定義。** §0 を改訂: 「フレームワークが決定・保証する」→「フレームワークのアーキテクチャが構造的に保証する」。焦点を「禁止ルール」から「API 設計上の保証」に転換。公開 API に `any` を含めない・エスケープハッチを提供しない・利用者がフレームワーク経由で型を破る手段を存在させない、の 3 原則を明記。§11 設計制約まとめも同内容に更新 |
| Ver.2.4 | 2026-04-11 | **型安全をフレームワーク実装方針として再定義。** §0: 型安全の主語を「開発者への制約」から「フレームワーク自身の実装が決定・保証する」に変更。Core 5 ファイルへの `any` / `@ts-ignore` / `@ts-expect-error` / `as any` 禁止を明記。「公開 API は型推論ファーストで設計し、利用者が型引数を省略できることを目標とする」を原則に追加。§11 設計制約まとめの型安全欄も同内容に更新 |
| Ver.2.3 | 2026-04-11 | **エラーハンドリング強化・バリデーター強化。** [エラーハンドリング] §5.4 に `ErrorResponse` / `HTTPError` 型定義を追加。§10 を全面改訂: エラーレスポンス形式を `ErrorResponse` に統一（§10.1）、405 Method Not Allowed ケース追加・`HTTPError` スロー時の動作定義（§10.2）、`HTTPError` 使用例追加（§10.3）、`onError` 優先順位の戻り値挙動を明確化（§10.4）。[バリデーター] §5.3 に `Rule` discriminated union 型定義・`RuleBase` 共通オプションを追加（`types.ts` へ移動）。§8.1 の Rule 表を更新: `nullable` 全 Rule 共通追加、`string` に `pattern` / `enum` 追加、`number` に `integer` 追加。バリデーション仕様に配列インデックス記法（`tags[0]`）・`nullable` 挙動を追記。§5.4（旧 §5.4 Route/Method）を §5.5 に繰り下げ |
| Ver.2.2 | 2026-04-11 | **組み込みミドルウェア（§8.1）を廃止。** §8.1 削除、バリデーターを §8.2 → §8.1 に繰り上げ、§8.3（予約済み）削除、§8 セクション見出し・`middleware.ts` 役割記述を更新 |
| Ver.2.1 | 2026-04-11 | **静的ファイル配信（§8.4）を廃止。** §8.4 削除、§2.1 の `Deno.readTextFile` 用途を `.env` 読み込みのみに修正、`deno.json` から `@std/path` 依存を削除、§3 `middleware.ts` の役割記述を更新 |
| Ver.2.0 | 2026-04-11 | **設計仕様書（Version 1.0 / 2025年）に基づき全面改訂。** アーキテクチャを刷新: ファイルベースルーティング廃止 → 明示的 Router API（§7）、5 ファイル Core 構成確定（§3）、Context 型変更（`ctx.request` → `ctx.req`、§5.1）、デュアルデプロイパターン明文化（§2.2）、バリデーター仕様整備（§8.2）、静的ファイル配信仕様整備（§8.4）、絶対原則を §0 として独立（`any` 型・`npm:` 禁止の不変規則）。Ver.1.x 系は設計仕様書制定前の暫定記載であり、本バージョンをもって正式仕様とする |
| Ver.1.30 | 2026-04-10 | 初版（暫定）: リリース計画ルールブックの記載を元に合成。§1〜§12 全体を策定（ファイルベースルーティング・`defineHandler`・各種組み込みミドルウェア・CLI 等）。設計仕様書との乖離があり Ver.2.0 にて全面改訂 |
