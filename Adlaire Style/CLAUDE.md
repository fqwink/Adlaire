# Adlaire Style — 開発規約

> **`RULEBOOK`（ルールブック / 仕様書）は絶対原則である。**
> すべての実装は RULEBOOK の仕様に基づいて行うこと。
> **RULEBOOK に記載のない機能を実装してはならない。**
> 新機能・変更は、**まず RULEBOOK に仕様を策定・記載してから実装に着手すること。**

---

## Adlaire 共通規約への準拠

本プロジェクトは、統合リポジトリルートの `CLAUDE.md`（共通規約）に完全準拠する。
共通規約と本ファイルが衝突する場合、**共通規約を優先する**。

## 技術規約

### 使用技術

- **純粋 CSS**（CSS カスタムプロパティ / CSS Nesting）
- **Deno / TypeScript**（`build.ts` ビルドスクリプトのみ）
- JavaScript フレームワーク・プリプロセッサ・PostCSS 等は使用しない

### 禁止事項

- `npm:` プレフィックスのインポートを全面禁止（共通規約）
- CSS 内の外部リソース参照（CDN・Google Fonts 等）禁止
- JavaScript によるスタイル操作（class 付与除く）禁止
- `!important` の多用禁止（やむを得ない場合はコメントで理由を明記）

### CSS 実装規則

- すべてのビジュアル値は CSS カスタムプロパティ（デザイントークン）経由で指定する
- ハードコードされた色・スペーシング・フォントサイズ値を CSS コンポーネントに直接記述してはならない
- セレクタ詳細度は可能な限り低く保つ（クラスセレクタ1段を基本とする）
- `!important` は原則禁止
