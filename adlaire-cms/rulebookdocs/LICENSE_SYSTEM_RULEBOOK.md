## 1. 基本宣言

### 1.1 位置づけ

本 RULEBOOK は、Adlaire Static CMS の API キー認証システムに関する恒常的規範文書である。
セキュリティ強化を目的とし、商業利用・非商業利用を問わず全利用者に適用される。

### 1.2 適用バージョン

- **Ver.3.0 以降**: API キー必須（未適用時は猶予期間後に動作停止）。
- **Ver.2.x 系**: API キーなしでも動作する（任意適用）。

### 1.3 上位原則

- Adlaire License Ver.2.0 の規定に準拠する。
- ARCHITECTURE_RULEBOOK.md の設計原則に従う。
- LIFECYCLE_SYSTEM_RULEBOOK.md のセットアップフローと整合する。

### 1.4 基本方針

- **セットアップ時に API キーは不要**とする。
- **初回ログインからの猶予期間は 3日間**とする。猶予期間超過後、未登録の場合は動作停止。
- **キーの登録・管理は管理画面で完結**する。利用者が公式サイトを訪問する必要はない。
- **1アカウント = 1キー**とする。複数キーの発行を禁止する。
- **メール認証は不要**とする。
- **共有サーバーで動作** — 特殊な PHP 拡張を要求しない。

## 2. API キー体系

### 2.1 キー構成（3層 + 商用拡張）

| キー | 生成元 | 生成タイミング | 役割 |
|------|--------|---------------|------|
| **システム固有APIキー** | ローカル自動生成（ランダム値） | 初回ログイン時 | インストール固有の識別子。公式サーバーへの登録トークン |
| **プライマリーAPIキー** | 公式サーバー発行 | 登録完了時 | 開発元（Adlaire Group）とシステム間の管理用キー |
| **セカンドAPIキー** | 公式サーバー発行 | 登録完了時 | 非商用利用の承認キー |
| **サードパーティーAPIキー** | 公式サーバー発行 | 商業ライセンス契約後 | 商用利用の承認キー（追加） |

### 2.2 利用形態とキー構成

| 利用形態 | 必要なキー |
|---------|-----------|
| 非商用利用 | システム固有 + プライマリー + セカンド |
| 商用利用 | システム固有 + プライマリー + セカンド + サードパーティー |

### 2.3 キー発行条件

| キー | 条件 | 費用 |
|------|------|:----:|
| システム固有APIキー | 初回ログイン時に自動生成 | 無料 |
| プライマリーAPIキー | 公式サーバーへの登録完了 | 無料 |
| セカンドAPIキー | 公式サーバーへの登録完了 | 無料 |
| サードパーティーAPIキー | 商業ライセンス契約の締結 | 有料 |

### 2.4 複数キーの禁止

- 1アカウント（1インストール）につき、各キーは 1つのみ発行する。
- 同一利用者が複数のキーセットを取得することを禁止する。

## 3. フロー

### 3.1 全体フロー

```
【Phase 0: セットアップ】
  bundle-installer.php → APIキー不要。通常通りインストール完了。

【Phase 1: 初回ログイン】
  管理画面にログイン
    → システム固有APIキーをランダム生成 → data/system/license.json に保存
    → 猶予期間 3日間を開始（初回ログイン日時を記録）
    → 管理画面に公式サーバー登録を促す通知を表示

【Phase 2: 登録（管理画面で完結）】
  管理画面 → ライセンス設定
    → システム固有APIキーを公式サーバー API に送信
    → 公式サーバーが登録処理
    → プライマリーAPIキー + セカンドAPIキー を受信
    → license.json に保存
    → 非商用利用開始

【Phase 3: 商用利用（契約後）】
  商業ライセンス契約完了
    → 管理画面 → ライセンス設定 → サードパーティーAPIキーを入力・登録
    → license.json に追加保存
    → 商用利用開始

【猶予期間超過】
  初回ログインから 3日以内に Phase 2 が完了しない場合
    → 管理画面・公開ページ・REST API のすべてが動作停止
    → ライセンス登録画面のみ表示
```

### 3.2 猶予期間

- **起算**: 初回ログイン日時（`license.json` の `first_login_at` フィールド）。
- **期間**: 3日間（72時間）。
- **超過時**: 全機能停止。管理画面のライセンス登録機能のみ利用可能。
- **登録完了後**: 猶予期間に関係なく即座に通常利用開始。

### 3.3 動作停止時の挙動

- HTTP ステータス 503 Service Unavailable を返す。
- 最小限の HTML でエラーメッセージとライセンス登録への導線を表示する。
- `bundle-installer.php` は動作を許可する（セットアップのため）。
- 管理画面のライセンス登録機能は動作を許可する（登録完了のため）。

## 4. システム固有APIキー仕様

### 4.1 生成

- **タイミング**: 初回ログイン時に自動生成。
- **方式**: 暗号論的擬似乱数（`random_bytes()`）によるランダム値。
- **形式**: 64文字の16進数文字列（32バイト = 256ビット）。
- **一意性**: ランダム値による衝突確率は無視できるレベル。

### 4.2 用途

- 公式サーバーへの登録時にインストールを一意に識別するトークンとして使用する。
- ローカルでの認証には使用しない（登録トークンのみ）。

---

## 5. プライマリーAPIキー仕様

### 5.1 役割

- 開発元（Adlaire Group）とインストール済みシステム間の管理用キー。
- 公式サーバーとの通信時に認証情報として使用する。

### 5.2 発行

- 公式サーバーがシステム固有APIキーの登録を受理した時点で発行する。
- セカンドAPIキーと同時に発行する。

---

## 6. セカンドAPIキー仕様

### 6.1 役割

- 非商用利用の承認キー。
- 本キーの存在により、当該インストールが非商用利用として承認されていることを示す。

### 6.2 発行

- プライマリーAPIキーと同時に発行する。
- 非商用利用者は本キーにより全機能を利用可能。

---

## 7. サードパーティーAPIキー仕様

### 7.1 役割

- 商用利用の承認キー。
- プライマリー + セカンドに**追加**して登録する（置き換えではない）。
- 本キーの存在により、当該インストールが商用利用として承認されていることを示す。

### 7.2 発行

- 商業ライセンス契約の締結後に公式サーバーから発行する。
- 管理画面のライセンス設定から入力・登録する。

### 7.3 Commercial 決済

- 決済方式は未定。決定後に本セクションを更新する。

## 8. ファイル構成

### 8.1 ライセンスデータファイル

- **パス**: `data/system/license.json`
- **権限**: 0600（owner のみ読み書き可）
- **形式**: JSON

```json
{
  "first_login_at": "2026-04-04T10:00:00+09:00",
  "system_key": "a1b2c3...（64文字hex）",
  "primary_key": "...",
  "second_key": "...",
  "third_party_key": "",
  "registered_at": "2026-04-04T10:05:00+09:00"
}
```

| フィールド | 必須 | 内容 |
|-----------|:----:|------|
| `first_login_at` | 必須 | 初回ログイン日時（ISO 8601）。猶予期間の起算点 |
| `system_key` | 必須 | システム固有APIキー（64文字hex） |
| `primary_key` | 登録後 | プライマリーAPIキー |
| `second_key` | 登録後 | セカンドAPIキー |
| `third_party_key` | 商用時 | サードパーティーAPIキー（空文字 = 未登録） |
| `registered_at` | 登録後 | 公式サーバー登録完了日時（ISO 8601） |

### 8.2 実装ファイル

| ファイル | 責務 |
|---------|------|
| `Core/license.php` | LicenseManager クラス（キー生成・検証・登録・管理） |

### 8.3 修正ファイル

| ファイル | 修正内容 |
|---------|---------|
| `index.php` | 起動ゲート追加（ライセンス状態チェック） |
| `Core/admin-ui.php` | ライセンス設定画面（登録・管理・サードパーティーキー入力） |
| `Core/api.php` | ライセンス関連 API エンドポイント追加 |

### 8.4 ARCHITECTURE_RULEBOOK との整合

- `Core/license.php` は Core ディレクトリに配置する（§2.2 準拠）。
- `data/system/license.json` は data/system/ に配置する（§3.1 準拠）。
- require 順序は `Core/helpers.php` の直後に `Core/license.php` を配置する。

## 9. 検証フロー

### 9.1 起動時ゲート

```
index.php 起動
    │
    ├─ install.lock 存在チェック
    │   └─ なし → bundle-installer.php（セットアップ）
    │
    ├─ license.json 存在チェック
    │   └─ なし → 初回ログイン処理へ（システム固有キー生成・猶予開始）
    │
    ├─ 登録状態チェック（primary_key が存在するか）
    │   ├─ 未登録 → 猶予期間チェック
    │   │   ├─ 猶予期間内（3日以内） → 通常起動 + 登録促進通知
    │   │   └─ 猶予期間超過 → 動作停止（ライセンス登録画面のみ）
    │   │
    │   └─ 登録済み → 通常起動
    │
    └─ 商用利用チェック（必要に応じて third_party_key 確認）
```

### 9.2 検証タイミング

- `index.php` の起動時に毎回実行する。
- 検証結果のキャッシュは行わない（改ざん防止）。

## 10. 管理画面 UI

### 10.1 ライセンス設定画面

管理画面内に「ライセンス」設定セクションを設ける。

**未登録状態:**
- システム固有APIキーの表示
- 「公式サーバーに登録」ボタン → API 呼び出し → キー受信・保存
- 猶予期間の残り日数表示

**登録済み状態:**
- ライセンス情報表示（登録日・キー種別・利用形態）
- キー更新ボタン
- サードパーティーAPIキー入力フィールド + 登録ボタン（商用利用向け）

### 10.2 通知

- 未登録時: 管理画面の全ページに登録促進バナーを表示する。
- 猶予期間残り1日: 警告レベルの通知に切り替える。

## 11. セキュリティ要件

### 11.1 ファイル保護

- `license.json` のファイル権限は 0600 とする。
- `license.json` への書き込みはアトミック操作（一時ファイル + rename）で行う。
- `license.json` のシンボリックリンク検出を行う。

### 11.2 通信保護

- 公式サーバーとの通信は HTTPS を必須とする。
- API レスポンスの改ざん検出を行う。

### 11.3 改ざん防止

- `license.json` の改ざん検出を行う（キー間の整合性検証）。
- 検証結果をキャッシュしないことで、ファイルの差し替えを即座に検出する。

## 12. 禁止事項

- API キーの共有・転売・公開を禁止する。
- API キーの偽造・改ざん・生成ツールの作成を禁止する。
- 検証ロジックのバイパス・無効化・改変を禁止する。
- 複数キーの取得を禁止する（1アカウント = 1キー）。
- 上記の違反は Adlaire License Ver.2.0 §8（ライセンスの終了）に基づき、
  全権利の自動終了事由とする。
